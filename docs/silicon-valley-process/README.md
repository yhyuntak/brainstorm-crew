## Quick Reference

### 핵심 철학
- **Ship Early, Ship Often** - 빠르게 배포하고 반복
- **Make it Work → Right → Fast** - 순서가 중요
- **Measure Everything** - 추측 말고 데이터
- **Fail Fast** - 실패할 거면 빨리
- **Done is Better Than Perfect** - 완벽보다 완료

### 팀 구조
- **Two-Pizza Team** (6-8명)
- Tech Lead + PM + Engineers (3-4) + Designer

### 타임라인 예시
- Week 1-2: 킥오프 + Sprint 0 (Foundation)
- Week 3-4: Core functionality
- Week 5-8: Alpha/Beta
- Week 9-12: Production hardening + GA
